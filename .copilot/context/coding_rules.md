# simple-ledger コーディングルール インデックス

このファイルは、3 つの詳細なコーディングルールドキュメントへのインデックスです。

---

## 📚 ドキュメント一覧

### 1. [プロジェクト概要](./project_overview.md)

プロジェクト全体の構成、技術スタック、セットアップ手順などをまとめています。

**含まれる内容:**

- プロジェクト説明
- 技術スタック（フロントエンド・バックエンド・インフラ）
- ディレクトリ構成
- セットアップ手順
- CI/CD 設定

### 2. [フロントエンド コーディングルール](./frontend_coding_rules.md)

Next.js・TypeScript・React に関連するコーディングルールと開発ガイドラインです。

**含まれる内容:**

- 基本設定（ESLint、TypeScript、pnpm）
- API クライアント層の共通化
- コンポーネント管理
- デザインシステム（デジタル庁準拠）
- コンポーネント開発ガイドライン
- 定数・型管理
- Button コンポーネント拡張ガイド
- コード生成・分析時のポイント

### 3. [バックエンド コーディングルール](./backend_coding_rules.md)

Go・Gin・GORM に関連するコーディングルールと開発ガイドラインです。

**含まれる内容:**

- 基本設定（Go 1.25.4、パッケージ管理、Linter）
- エラーハンドリング ルール（必須）
- 環境変数設定
- CRUD 実装パターン
- JWT 認証パターン
- テスト方針
- コード生成時のガイドライン
- コード分析時のポイント

## 🔍 用途別ガイド

### フロントエンド開発を始める時

1. [プロジェクト概要](./project_overview.md) - セットアップ手順を確認
2. [フロントエンド コーディングルール](./frontend_coding_rules.md) - 全体を読み、特にコンポーネント開発ガイドラインを理解
3. 必要に応じて[デザインシステム](./frontend_coding_rules.md#%EF%B8%8F-デザインシステムデジタル庁準拠)を参照

### バックエンド開発を始める時

1. [プロジェクト概要](./project_overview.md) - セットアップ手順を確認
2. [バックエンド コーディングルール](./backend_coding_rules.md) - 全体を読む、特にエラーハンドリングと CRUD パターンに注意
3. 新しい機能を追加する時は[CRUD 実装パターン](./backend_coding_rules.md#crud-実装パターンuser-モデルに従う)を参照
