# 回答ルール

このプロジェクトの作業中、常に以下のように応答します。

- すべての返答は日本語で行う。
- 技術的な説明は、適切なコード参照や文書へのリンクを含めて丁寧に述べる。
- 回答が複数ステップあるときは、箇条書きで整理し、必要に応じて順序を示す。

## ファイル作成時のルール

### Go 言語ファイル作成時

1. **package 宣言は 1 回のみ**

   - package 文が重複しないことを確認する
   - 複数のファイルを同時に作成する場合は、各ファイルの内容を最後に検証する

2. **複数ファイルの同時作成**

   - create_file ツールを複数回使用する場合は、各ファイルの内容が独立していることを確認
   - 必要に応じてターミナルコマンドで `head -1` などで検証する

3. **作成後の検証**

   - ファイルサイズが異常に小さくないか確認
   - `go build` でコンパイルエラーがないか確認
   - テストファイルの場合は `go test` で実行

4. **破損したファイルの対応**
   - ファイルが破損している場合は、即座に `rm` で削除してから `create_file` で再作成する
   - 削除前に必ずファイルの実サイズを確認すること

### テストファイル作成時

- package 宣言の直後に余計な行がないことを確認
- import 文が正しくグループ化されている
- 不要なインポートを含まない
- モック構造体の定義が完全であることを確認
